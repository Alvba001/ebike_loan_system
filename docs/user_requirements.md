# User Requirements

## 1. Borrower (User)
The Borrower (likely a student or individual) is the primary user applying for an electric bike loan.

*   The Borrower shall be able to **register** for an account using their name, email, and password.
*   The Borrower shall be able to **log in** to their account.
*   The Borrower shall be able to **view their dashboard** which displays loan status and repayment information.
*   The Borrower shall be able to **apply for a loan** by selecting a bike model, loan duration, and providing personal details (NIN, Purpose).
*   The Borrower shall be able to **upload required documents** (ID Card, Utility Bill, Guarantor Document, Supporting Document) during the loan application.
*   The Borrower shall be able to **view the estimated monthly payment** before submitting the application.
*   The Borrower shall be able to **make a repayment** for an approved loan.
*   The Borrower shall be able to **view their repayment history**.
*   The Borrower shall be able to **receive notifications** (e.g., via dashboard or implied system logic) regarding loan status or reminders.

## 2. Admin
The Admin manages the system, loans, and inventory.

*   The Admin shall be able to **log in** with administrative credentials.
*   The Admin shall be able to **view a dashboard** with an overview of system activities.
*   The Admin shall be able to **add new bikes** to the inventory by providing a serial number and model.
*   The Admin shall be able to **view all loan applications**.
*   The Admin shall be able to **view detailed information** of a specific loan application, including uploaded documents.
*   The Admin shall be able to **approve a loan application**, which automatically generates a repayment schedule.
*   The Admin shall be able to **reject a loan application**, which notifies the user.
*   The Admin shall be able to **assign a specific bike** (from available inventory) to an approved loan.
*   The Admin shall be able to **view all repayment records**.
*   The Admin shall be able to **trigger checks for repayment reminders**, which generates notifications for users.
